HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Only use on H1 the header and only use it once
  • Everything will go from left to right, top to bottom
  • "src=" is for images, "href=" is for links
  • Use "link href="'Address of your css file'" rel="stylesheet" type="css/text"
    to link the stylesheet to the website

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Class can be used for multiple elements
  • Each id should only be used for once per page
  • The "Absolute" position puts an element in the defined position relative to it's parent element
  • The "Relative" position puts an element in the same place as it would by default but lets you move it from there.

Git

  • git status: | Checks what branch we are currently on
  • git checkout -b "branch-name" | Creates a new branch and switches to it
  • git checkout "destionation branch" | Changes the active working branch
  • git commit -m | Is to add a commit to your local git repository with a message
  • git init | Creates the files need to make a directory into a repository

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Quite similar to python
  • "for" and "if" are loops
  • use "function" to create a function
  • use var to create a variable
